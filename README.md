# 🚀 FastAPI User Management API

Простой и мощный пет-проект на **FastAPI** для управления списком пользователей. Реализовано асинхронное взаимодействие с **PostgreSQL**, контейнеризация через **Docker** и встроенный веб-интерфейс.

## ✨ Особенности
- **⚡ Асинхронность**: Неблокирующий код на базе `asyncpg` для максимальной производительности.
- **📈 Мониторинг**: Middleware для логирования "медленных" запросов (выполнение > 700мс).
- **🛡 Валидация**: Строгая проверка данных (email, возраст, длина строк) через Pydantic.
- **🖥 Frontend**: Встроенная одностраничная панель управления (HTML/JS) с пагинацией.
- **📊 Database**: Автоматическая инициализация таблиц и индексов при старте.
- **📈 Статистика**: Эндпоинт `/stats` и виджет на фронтенде (количество пользователей, средний возраст, самый старший/младший).

---

## 🛠 Технологический стек
- **Backend**: FastAPI, Uvicorn
- **Database**: PostgreSQL 15
- **Infrastructure**: Docker, Docker Compose, Neon.tech (Cloud DB)

---

## 📋 Эндпоинты (API)
- `GET /` — Главная страница (Frontend)
- `GET /health` — Статус работоспособности API
- `GET /stats` — Статистика по пользователям
- `GET /users` — Список пользователей с пагинацией (`limit`, `page`)
- `POST /add_user` — Создание нового пользователя
- `DELETE /delete_user` — Удаление пользователя (нужен админ-пароль)
- `DELETE /clear` — Полная очистка таблицы (нужен админ-пароль)

## 🚀 Запуск проекта

### 1. Настройка окружения
Создайте файл `.env` в корне проекта:
```env
DB_USER=postgres
DB_PASSWORD=mysecretpassword
DB_NAME=user_db
ADMIN_PASSWORD=superadmin
DATABASE_URL=postgresql://postgres:mysecretpassword@db:5432/user_db
```

### 2. Управление через Docker Compose


| Команда | Действие |
| :--- | :--- |
| `docker-compose up --build` | Сборка и запуск проекта (логи в консоли) |
| `docker-compose up -d` | Запуск в фоновом режиме (демон) |
| `docker-compose logs -f api` | Просмотр логов приложения |
| `docker-compose stop` | Остановка контейнеров |
| `docker-compose down -v` | Полное удаление контейнеров и данных БД |

Приложение доступно по адресу: **[http://localhost:8000](http://localhost:8000)**  
Интерактивная документация: **[http://localhost:8000/docs](http://localhost:8000/docs)**
